
## 第一章

CPU：计算机的“大脑”，负责执行指令、进行数据运算和控制整个系统的协调工作，是运算和控制的核心。

ALU：CPU的核心运算单元，专门处理算术运算和逻辑运算。

GPRS：CPU内高速存储单元，用于临时存放运算数据、中间结果和地址，速度远快于内存，可提升处理效率。

PC：CPU内的“导航仪”，始终存储下一条要执行的指令在内存中的地址，确保程序按顺序运行。

IR：临时存放CPU当前正在解码和执行的指令，是指令从内存到执行单元的“中转站”。

CU：CPU的“指挥官”，负责解读指令，向计算机其他部件（如ALU、内存）发送控制信号，协调各部件同步工作。

主存储器（内存）: 是CPU能直接访问的临时存储区域，用于存放当前运行的程序和数据，断电后数据会丢失。

总线：计算机各部件之间传输数据、地址和控制信号的“高速公路”，分为数据总线、地址总线和控制总线。

机器指令：计算机能直接识别和执行的二进制代码（0和1），是最低级的指令形式，每一条对应一个具体操作。

汇编语言：面向机器的低级编程语言，用英文助记符替代机器指令的二进制代码，需通过汇编程序翻译成机器指令才能执行。

1.冯·诺依曼计算机组成及功能

运算器：执行算术运算和逻辑运算。

控制器：解读指令，向其他部件发送控制信号，协调整机同步工作。

存储器：存储程序和待处理的数据。

输入设备：向计算机输入数据和指令。

输出设备：将计算机处理结果反馈给用户。

2.“存储程序”工作方式

将待执行的程序和相关数据预先存入计算机存储器，计算机在控制器控制下，按地址顺序从存储器中取出指令并依次执行，实现自动连续运算。

3.机器指令：能直接执行，是计算机可识别的二进制代码。

汇编指令：不能直接执行，需通过汇编程序翻译成机器指令后才能执行。

4.程序开发与运行阶段及对应文件

1. 编辑阶段：使用计算机语言编写源代码，得到源程序文件（如.cpp 、.java文件）。

2. 编译/汇编阶段：将源程序转换为机器指令，得到目标文件（如.obj、.o文件）。

3. 链接阶段：将目标文件与库文件等整合，得到可执行目标文件（如.exe、.out文件）。

4. 运行阶段：加载可执行文件到内存，CPU执行指令，无新文件生成，仅产生运行结果。

5. 一个高级语言程序被转换为可执行目标文件后，存储在外存储器中，如硬盘、U盘等。

6. 一个可执行目标文件被启动并执行后，其中的指令和数据会加载到主存储器中，CPU从内存中读取指令和数据进行运算。

7. 指令中地址码字段用于指出操作数的存储地址，或下一条要执行指令的地址

位地址码字段的位数与计算机主存储器的地址空间大小有关，地址码位数越多，可访问的内存地址范围越大。

## 第二章

1. 为什么计算机内部采用二进制表示信息?既然计算机内部所有信息都用二进制表示，为什么还会用到十六进制?

采用二进制的原因：计算机内部采用二进制，主要是因为二进制系统在物理上最容易实现和稳定。这种表示方式抗干扰能力强，可靠性高，运算规则简单。

还使用十六进制的原因：虽然计算机内部使用二进制，但二进制数位长、不易读写和记忆。十六进制与二进制有天然的进制转换便利，一个十六进制位可以精确地表示4个二进制位。

2.常用的定点数编码方式有哪几种?它们通常用来表示什么?

1.无符号数：所有二进制位都用来表示数值本身，没有符号位。它通常用来表示非负整数，如内存地址、数组索引等。

2.原码：最高位为符号位（0表示正，1表示负），其余位表示数值的绝对值。

3.补码：最高位同样是符号位，但负数的表示方式与原码不同。它可以将减法运算统一为加法运算，简化了计算机内部运算器的设计。

3.为什么现代计算机中大多用补码表示带符号整数?

统一加减法运算：使用补码，可以将减法运算转换为加法运算。即 A - B 可以转换为 A + (-B)，其中 -B 是 B 的补码。这样，计算机的运算器只需要设计一个加法器，就可以同时完成加法和减法运算，大大简化了硬件电路的设计。而且还可以统一0的表示方法。

4.在浮占数的基教和总位数一定的情况下， 浮点数的表示范围和精度分别由什么决定?两者如何相互制约?

表示范围：主要由阶码（指数）的位数决定。阶码的位数越多，所能表示的指数范围就越大，从而浮点数能表示的范围就越广。

精度：主要由尾数（有效数字）的位数决定。尾数的位数越多，能够表示的有效数字位数就越多，数值的精度就越高。

相互制约：在总位数固定的情况下，分配给阶码和尾数的位数是此消彼长的关系。如果增加阶码的位数以扩大表示范围，就必须减少尾数的位数，这将导致精度下降。反之，如果增加尾数的位数以提高精度，就必须减少阶码的位数，这将导致表示范围缩小。因此，浮点数的设计需要在表示范围和精度之间进行权衡。

## 第三章

1.为何在高级语言和机器语言中都要提供按位运算? 为何高级语言需要提供逻辑运算? 按位运算和逻辑运算的差别是什么?

在高级语言和机器语言中都提供按位运算，是因为计算机底层的数据以二进制位bit形式存储和处理。按位运算能够直接对每一位bit进行操作，效率高，常用于底层控制、寄存器操作、掩码处理等需要精确控制二进制位的场合。

高级语言提供逻辑运算是为了表达程序中的逻辑关系和判断条件，如“与”“或”“非”等，用以实现分支、循环等控制结构，便于程序设计和逻辑推理。

按位运算和逻辑运算的主要区别在于：

按位运算：以二进制位为单位操作，作用于整数类型的每一位，处理的是数值的位模式

逻辑运算：以布尔逻辑为基础，作用于逻辑值（真或假），处理的是条件的真假。

2.如何进行逻辑移位和算术移位? 它们各施加于哪种类型的数据?

逻辑移位

操作方式：将数值的所有二进制位整体左移或右移，空出的位以 0 填充。

应用对象：主要用于无符号数。

算术移位

操作方式：左移与逻辑左移相同；右移时保持符号位（最高位）不变，空出的位以符号位原值填充。

应用对象：主要用于 有符号数。

3.移位运算和乘除运算有什么关系?

左移运算 <<：相当于乘以 2 的n次方。

右移运算 >>：相当于除以 2 的n次方取整

4.高级语言中的运算和机器语言(即指令)中的运算是什么关系? 假定某一个高级语言源程序P中有乘除运算，但机器Ｍ中不提供乘除运算指令，则程序P能否在机器M上运行? 为什么?

高级语言中的运算与机器语言（即机器指令）中的运算存在抽象与实现的关系：

高级语言运算是对计算过程的抽象描述，如加、减、乘、除、逻辑运算等，强调表达式的逻辑意义；

机器语言运算是由处理器提供的具体操作指令，用于在硬件层面实现这些计算。高级语言中的每一种运算，最终都要通过编译器翻译为一系列机器指令来实现。

若机器 M 不提供乘除指令，而高级语言程序 P 中包含乘除运算，则程序 P 仍然可以在机器 M 上运行。

编译器或解释器会将乘除运算转换为一系列等价的基本指令（如加、减、移位、循环累加等）来模拟实现；

只要机器 M 提供足够的基本运算能力（加减、移位和比较），乘除等复杂运算均可由软件算法实现。

## 第四章

指令集体系结构 (ISA)：ISA 是软件和硬件之间的抽象接口（契约），它定义了处理器能理解的所有指令集合、寄存器结构、内存模型以及异常处理机制，是程序员能看到的机器逻辑视图。

指令：计算机执行操作的最小功能单位，它是一串二进制代码，明确告诉处理器要做什么动作（如加法、跳转）以及对谁（哪些数据）进行操作。

操作码：指令中用于指定具体操作类型的字段，例如告诉 CPU 是执行“加法 (ADD)”、“加载 (LOAD)”还是“跳转 (JUMP)”。

地址码：指令中用于描述操作数位置的字段，它指明了数据是存放在寄存器中、内存地址中，还是作为立即数直接给出，或者指明运算结果的存放地址。

立即寻址：操作数直接包含在指令本身中，不需要去访问内存或寄存器，速度最快。

直接寻址：指令中直接给出了操作数在内存中的有效物理地址，CPU 直接根据这个地址去内存抓取数据。

间接寻址：指令中给出的地址不是数据本身的地址，而是存放数据地址的地址，需要访问两次内存才能拿到数据。

寄存器寻址：操作数存放在 CPU 内部的通用寄存器中，指令中只需给出寄存器的编号，无需访问内存，速度非常快。

寄存器间接寻址：寄存器中存放的不是数据本身，而是数据在内存中的地址，相当于把寄存器当作指针来使用，通过它指向内存去获取数据。

帧指针：一个专门的寄存器，始终指向当前函数栈帧的一个固定基准位置（通常是栈帧底部），主要用于方便地通过偏移量访问函数的参数和局部变量。

栈帧：函数在被调用时，在栈内存中分配的一段连续区域（活动记录），用于保存该函数的返回地址、传入参数、局部变量以及保存的寄存器上下文，函数返回时该区域会被释放。

ABI 规范：应用程序二进制接口，它定义了程序在二进制层面如何与操作系统及硬件交互的规则（包括函数调用约定、参数如何传递、栈怎么对齐、系统调用方式等），确保不同编译器生成的二进制代码能互相链接和运行。

1.一条指令中应该显式或隐式地给出哪些信息？

一条完整的指令必须告知计算机操作码、操作数地址，以及下一条指令的地址，从而构成完整的执行逻辑

2.RISC 处理器的特点有哪些？

RISC的核心特点是指令数量少且功能简单，大部分指令在一个时钟周期内完成；指令格式固定且长度一致，易于硬件译码和流水线优化；采用 Load/Store 结构（只有加载/存储指令访问内存，运算指令只在寄存器间进行）；并且通常配备大量的通用寄存器以减少访存频率

3.按值传递参数和按地址传递参数两种方式有哪些不同之处？

主要区别在于数据及其副作用：按值传递是将实参的值复制一份给形参，函数内修改形参不影响外部实参，适合小数据但开销随数据变大而增加；按地址传递是将实参的内存地址传给形参，函数通过地址直接操作原数据，修改会改变外部实参，传递大对象时效率极高（只需传一个指针），但安全性相对较低

4.为什么数据在存储器中最好按对齐方式存放？

主要原因是为了提高访存性能和简化硬件设计。CPU 通常按字或块为单位读取内存，如果数据对齐存放，CPU 只需一次访存即可取出数据；若数据不对齐，CPU 需要进行两次访存并进行拼接处理，严重降低效率，甚至在某些架构下会直接引发硬件异常

## 第五章

链接：将各种代码和数据片段（如不同的源文件编译后的结果）收集并组合成一个单一的、可加载到内存并执行的文件的过程

可执行目标文件：一种包含完全链接的机器代码和数据的二进制文件，操作系统可以直接将其加载到内存中并运行

符号解析：链接器将代码中对符号（函数或变量）的每一次引用，都与该符号的一个确定的定义关联起来的过程

重定位：链接器为程序中的指令和数据分配最终的运行时内存地址，并修改代码中对这些地址的引用，使其指向正确位置的过程

ELF 目标文件格式：Unix/Linux 系统中广泛使用的一种标准二进制文件格式，用于规范地存储目标代码、可执行文件和共享库的结构

节头表：ELF 文件中的一张索引表，用于描述文件中各个“节”（如代码节 .text、数据节 .data）在文件中的位置、大小和属性

全局符号：在当前模块中定义，并可以被其他模块引用（访问）的非静态函数或非静态全局变量

静态库：一组相关的目标文件（.o 文件）的打包集合，在链接时，链接器只从中提取被程序引用的部分并整合到最终的可执行文件中

符号的引用：在代码中实际使用某个变量或调用某个函数的地方，链接器需要根据这个引用去寻找其对应的定义地址

重定位信息：编译器生成的数据（通常存储在 .rel 节中），用于告诉链接器在合并文件时，应该如何修改代码或数据中的地址引用

1.从源程序到可执行机器代码的转换过程

这个过程通常由编译器驱动程序引导，分为四个主要阶段：预处理（将 #include 和宏展开）、编译（将 C 代码翻译成汇编语言）、汇编（将汇编代码翻译成机器指令，生成可重定位目标文件 .o），最后是链接（将多个 .o 文件及必要的库文件合并，解析符号引用并重定位地址），最终生成可执行目标文件

2.引入链接的好处

链接的主要好处是实现了模块化和效率提升。它允许程序员将大型软件分解为更小、更易管理的独立模块（文件），这不仅便于多人协作开发，而且支持分离编译——当修改某个模块时，只需重新编译该模块并重新链接，而无需重新编译整个应用程序，从而大大节省了时间和存储空间。

3.可重定位目标文件与可执行目标文件的主要差别

可重定位目标文件中的代码和数据地址通常从 0 开始，包含未解析的符号引用，不能独立运行；而可执行目标文件是完全链接好的，其中的代码和数据已被分配了最终的运行时绝对内存地址，符号引用已被解析，操作系统可以直接将其加载到内存中执行

4.静态链接器主要完成的两方面工作

静态链接器主要完成两大任务：符号解析，即把程序中每个符号的引用与该符号在某个输入文件中的唯一定义关联起来；以及重定位，即将多个输入文件的同类节合并，并为每条指令和全局变量分配具体的运行时内存地址，修正代码中对这些地址的引用

5.ELF各节主要包含的信息

.text 节：存放已编译程序的机器代码

.rodata 节：存放只读数据，如 printf 语句中的格式串、switch 语句的跳转表以及用 const 修饰的全局变量

.data 节：存放已初始化的全局变量和静态C 变量

.bss 节：存放未初始化（或初始化为 0）的全局变量和静态 C 变量，该节在目标文件中不占据实际磁盘空间，仅作为占位符

## 第六章

指令周期 ：CPU 从内存中取出一条指令并执行完这条指令所需的全部时间

数据通路：CPU 中负责数据存储、处理和传送的路径和部件集合（如 ALU、通用寄存器、内部总线等）

控制单元：CPU 的“指挥中心”，负责分析指令并发出控制信号，协调数据通路各部件按序工作

指令寄存器：CPU 中专门用于存放当前正在执行的那条指令的寄存器

指令译码器：控制单元的一部分，负责将指令寄存器中的操作码（Opcode）翻译成对应的控制电位

控制信号：由控制单元发出的电脉冲信号，用于告诉硬件部件（如 ALU 或内存）具体执行什么动作

中断服务程序：当计算机响应中断请求时，暂停当前程序转去执行的、专门处理该特定事件的程序

断点：调试程序时在代码中设置的一个位置，程序运行到此时会暂停（通常通过产生异常/陷阱实现），以便检查系统状态

指令流水线：一种将指令执行过程分解为多个子阶段（如取指、译码、执行），让多条指令在不同阶段并行重叠处理的技术

流水线深度：流水线被分割成的独立处理阶段（Stage/级）的数量

指令吞吐量：在单位时间内，流水线能够完成执行的指令总数量

1.CPU 的基本组成和基本功能

CPU 主要由数据通路和控制单元两大部分组成；其基本功能是循环执行“取指、译码、执行、访存、写回”等步骤，即从存储器中取出指令，解释指令含义，并控制计算机各部件进行相应的数据运算或逻辑操作

2.取指部件的功能

取指部件主要负责根据程序计数器（PC）中存放的地址，从指令存储器中读出当前需要执行的指令并送入指令寄存器，同时自动计算并更新 PC 的值，让 CPU 指向下一条将要执行的指令

3.控制器的功能

控制器是 CPU 的指挥中心，它负责对指令寄存器中的操作码进行译码分析，并根据当前指令的要求，向数据通路中的寄存器、ALU、多路选择器和存储器等部件发出精确的控制信号，协调它们按序工作

4.单周期处理器的 CPI、时钟周期与性能

单周期处理器的 CPI恒定为 1；其时钟周期长度必须由执行时间最长的那条指令决定；其性能较差是因为时钟频率被最慢的指令“拖累”变得很低，导致执行那些本可以很快完成的简单指令时浪费了大量时间

5.多周期处理器的设计思想与 CPI

多周期处理器的设计思想是将指令的执行过程切分成多个更小的步骤，每个步骤占用一个较短的时钟周期，允许复用硬件资源；在这种设计下，每条指令的 CPI 不相同，简单的指令占用较少的周期，复杂的指令占用较多的周期，从而提高了整体效率

## 第七章

随机存取存储器：一种允许在恒定时间内对任意存储单元进行读写访问的半导体存储器，其访问时间与数据所在的物理位置无关。

只读存储器：一种预先写入数据且在正常操作下仅能读取的非易失性半导体存储器，常用于存储系统引导程序或固件。

易失性存储器：一种必须持续通电才能维持数据存储状态的存储介质，一旦断电其内部数据即刻丢失。

程序访问局部性：程序在执行过程中呈现出的倾向于频繁访问局部内存区域的特性，具体表现为时间局部性（重复访问）和空间局部性（邻近访问）。

动态 RAM：利用电容内存储电荷的多少来代表二进制比特位的随机存取存储器，因存在漏电现象需定期进行刷新操作。

磁盘平均存取时间：磁盘完成一次读写请求所需的统计平均时间，在数值上等于平均寻道时间、平均旋转延迟与数据传输时间之和。

磨损平衡：固态硬盘控制器采用的一种算法机制，旨在将擦写操作均匀分布到闪存的所有物理块上，以防止个别块过早损坏从而延长设备整体寿命。

高速缓存：位于中央处理器与主存储器之间的小容量、高速度缓冲存储器，旨在缓解 CPU 高速处理能力与主存低速存取之间的性能瓶颈。

主存块：主存储器与高速缓存之间进行信息交换和传送的固定长度的基本数据单元。

Cache 行：高速缓存中用于存放一个主存块及其标记、有效位等控制信息的最小物理存储单元。

Cache 冲刷：将高速缓存中的内容清空或强制将脏数据写回主存的操作，以确保多级存储体系间的数据一致性。

命中率：CPU 发出的访存请求中，能在高速缓存中直接找到所需数据的次数占总访存请求次数的比率。

直接映射：一种将主存块映射到 Cache 中唯一固定行位置的映射方式，具有硬件电路简单但冲突概率较高的特点。

全相联映射：允许主存块装入 Cache 中任意可用行位置的映射方式，具有灵活性高、冲突率低但硬件比较电路复杂的特点。

组相联映射：将 Cache 划分为若干组，主存块映射到固定组但在组内可任意存放的折中映射方式，兼顾了灵活性与硬件成本。

1.计算机内部为何要采用层次化存储体系结构？层次化存储体系结构一般由哪几部分构成？

采用层次化结构是为了解决存储器在容量、速度和价格三者之间的矛盾，利用程序访问的局部性原理，将不同特性的存储器结合，使系统整体性能接近高速存储器，而容量和价格接近低速存储器。该体系结构从上至下一般由CPU 寄存器、高速缓存、主存储器和辅助存储器构成。

2.为什么在 CPU 和主存之间引入 cache 能提高 CPU 访问效率？

这是基于程序访问的局部性原理（时间局部性和空间局部性）。Cache 利用该原理，将 CPU 近期极可能频繁使用的数据和指令副本从低速的主存预先调入高速的 Cache 中；当 CPU 需要数据时，大部分情况下可以直接从高速 Cache 获取，从而大幅减少了 CPU 等待主存存取的时间，降低了系统的平均访存时间。

3.什么是 cache 映射的关联度？关联度与命中率、命中时间的关系是怎样的？

关联度是指在 Cache 映射机制中，主存中的一个块可以映射到 Cache 中不同行的候选位置数量。一般而言，关联度越高，命中率越高，但同时所需的硬件比较电路越复杂，导致判断是否命中的延迟增加，即命中时间变长（速度变慢）。

4.为什么要考虑 cache 的一致性问题？读操作时是否要考虑 cache 的一致性问题？为什么？

考虑一致性是为了确保数据在 Cache 和主存中保存的多个副本始终保持同步，防止 CPU 使用过期的“脏数据”导致计算错误。读操作时本身不需要执行一致性维护动作（如写回或更新），因为读取不会改变数据状态，不会产生新的不一致；一致性问题的根源在于写操作，只有当数据被修改时，才导致副本间产生差异，从而需要机制来修复这种不一致。

5.为什么程序员需要了解 cache 的结构和工作原理？

了解 Cache 原理（特别是局部性原理）能帮助程序员编写出对硬件友好的高效代码。例如，在遍历二维数组时，若按照数据在内存中的物理存储顺序进行访问，能最大化利用空间局部性，提高 Cache 命中率；反之，若代码编写不当导致频繁的 Cache 失效，程序的运行速度可能会出现断崖式下跌。

## 第八九章

中断：来自 CPU 外部（如 I/O 设备或时钟）的异步信号，强制 CPU 暂停当前正在执行的指令流，转而去执行相应的中断服务程序。

异常：由 CPU 内部执行指令过程中产生的同步事件，导致程序控制流的改变。

进程的上下文：操作系统在进行进程切换时必须保存和恢复的该进程运行所需的全部环境信息，包括程序计数器、寄存器堆数据及虚拟内存映射等。

故障：一种可被操作系统修复的异常类型，在处理程序解决了故障原因后，CPU 会重新执行引起故障的那条指令。

自陷：程序通过特定指令预先安排并主动触发的异常，通常用于从用户态请求操作系统内核的服务。

中断向量表：存储在内存特定位置的一张索引表格，用于保存各个中断源对应的中断服务程序的入口地址。

总线：计算机系统中连接多个功能部件并用于传输数据、地址和控制信号的公共共享通信通道。

串口：一种输入输出接口标准，通过数据线按比特位的顺序逐位进行数据传输，适用于远距离或低速通信。

设备编址：CPU 对 I/O 设备进行识别和访问的方式，通常通过给设备的寄存器分配独立的 I/O 端口地址或将其映射到物理内存地址空间来实现。

1.简述CLINT中断的处理流程和适用场景

CLINT主要负责处理CPU 核内的局部中断。其处理流程是：当计时器寄存器的值达到比较寄存器设定的阈值，或者软件向特定寄存器写入指令时，CLINT 会直接向其绑定的 Hart触发中断信号，CPU 随即跳转至中断向量表处理。它主要适用于生成高精度的计时器中断（如操作系统的任务调度时钟滴答）以及多核系统中的核间通信。

2.简述PLIC中断控制器的处理流程和适用场景。

PLIC是平台级中断控制器，用于管理所有外部设备的中断。其处理流程包含仲裁、分发、响应三个阶段：PLIC 首先汇集来自各个外设的中断线，根据预设的优先级仲裁出当前最高优先级的请求，将其发送给目标 Hart；Hart 响应中断后，需通过读取 Claim 寄存器获取中断号，处理完毕后再写 Complete 寄存器通知 PLIC 结束。它适用于管理大量外部 I/O 设备产生的全局中断。

3. 简述CPU的三种特权模式

为了实现操作系统与应用程序的隔离保护，CPU 设计了三种权限递减的模式：机器模式是最高特权级，可直接访问所有硬件资源，通常用于运行固件或 Bootloader；监管者模式权限次之，支持虚拟内存管理，主要用于运行操作系统内核；用户模式权限最低，无法直接访问硬件设备，仅用于运行普通的应用程序。

4.简述访问外设的三种方式。

CPU 与外设交互主要有三种机制：轮询方式下，CPU 需不断循环检查设备的状态寄存器以判断是否就绪，效率较低但硬件简单；中断方式下，设备就绪时主动向 CPU 发送信号，CPU 暂停当前任务转而处理设备请求，实时性较好；直接存储器访问（DMA）方式下，由专用 DMA 控制器接管总线，直接在设备与内存间传输数据块，CPU 仅需在开始和结束时参与，极大减轻了 CPU 负担，适合大数据量传输。
