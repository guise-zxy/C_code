## mkdir,touch
从文件系统的角度出发：
首先我们得有文件对吧！
因此，我们就可以联想到先"创造"文件或者文件夹的操作:
```

mkdir [-p] 文件夹路径 ：创建文件夹（目录）

[-p]:想要一次性创建2个以上的文件夹时添加


touch 文件路径：创建文件 




```

## cat,more
那现在有了文件或者文件夹这些操作对象，我们是不是就可以想着先"查看"文件里的内容：
```

cat 文件路径：
仅限查看文件内容；
注意：若文件内容为空，则什么都没有


more 文件路径：
仅限查看文件内容，文件内容太多时，可以以翻页的形式在命令行显示内容。"命令行"可以给我们提供2个操作:
    "空格---翻页
    "Q"----退出

```


## cp,mv,rm,通配符
接着，又因为，我们的参考对象是文件或者文件夹；
因此，对于文件或文件夹之间就可以进行"复制,移动,删除"操作：
```
cp:

cp [-r] 被复制的文件路径1 复制到的新文件(文件夹)的路径2：

-------------------------------------------------------------------------
路径1         路径2           结果
-------------------------------------------------------------------------
文件          文件夹          若这个文件夹存在，则将指定文件复制一份放到里面；若不存                               在，则复制一份内容与指定文件相同的没有后缀的文件
---------------------------------------------------------------------------                      
文件1           文件2         若文件1存在，文件2存在，则将文件1文件里的内容覆盖到文                               件2。若文件1存在，文件2不存在，则在指定路径下创建一                                个"同名"的内容与文件1内容相同的文件。
                             若文件1不存在，直接报错:cp: cannot stat 'rty.txt':                               No such file or directory
                             
---------------------------------------------------------------------------
文件夹         文件            不加[-r],则报错。加了[-r]也报错，因为不能把文件夹的                              所有内容复制到一个"文件"         
                             
                              
--------------------------------------------------------------------------
文件夹1          文件夹2           加[-r]，若文件2存在，则将文件夹1复制一份到文件夹                                   2。若文件2不存在，则创建一份"同名"文件夹，并将文                                  件夹1复制一份到其下
---------------------------------------------------------------------------







mv 文件/夹路径1 文件/夹路径2：可用来给文件改名
注意：
若都存在，没加任何选项，则路径2的被删除，路径1改名为路径2.
若，
若路径2不存在，则对文件路径1所指的文件/夹进行"改名操作"
详情见：





rm [-f -r] 文件/文件夹路径1     2   3    ······N：仅能删除文件
-f:删除文件夹时，需加上
-f:加上，则表示"强制删除"。就是删除某些高权限的文件/夹时，不会弹提示。直接删除(最好别使用)



通配符*：

```



## which查找命令本体文件，find2种模式查找文件内容

```
which Linux命令
:
查找Linux命令本体文件，返回文件路径
```

```
find 起始路径 -name "文件名(可配合通配符使用)":


find 起始路径 -size +或-n[kMG]：
查找大于n或小于n[kMG]的文件，返回路径
```



## grep,管道符|

```
(其他命令的结果可配合管道符"|"作为输入内容|)grep [-n] "关键词" 文件路径(不加则配合管道符"|"提供的输入流)：
:
过滤出指定文件中含"关键字"的行，加[-n]则显示行数。

```



##  wc，管道符|
```
(其它命令结果可配合管道符"|"作为输入内容|)wc [-c-w-l-m] 文件路径
：统计对应文件中的"单词数量，行数，字节数"
-c:字节数
-w:统计单词数量
-l:行数
-m:字符数

```

## tail

```
tail [-num  -f]  文件路径
```
：
追踪文件的尾部内容
-num：指定跟踪尾部的几行；不指定，则默认10行
-f：持续追踪。Ctrl+C强制退出





## 重定向符
```

>:将左侧指令的结果覆盖写入到右边文件
>>：将左侧指定的内容追加写入到右边文件

```
